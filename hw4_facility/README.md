
## Запуск

```bash
./run.sh data/fl_100_1
```

## Описание

### Жадник

Сортируем клиентов по убыванию спроса, каждого назначаем к самому дешевому объекту (расстояние + штраф за открытие, пропорциональный спросу).

Добавил k ближайших объектов для каждого клиента (ограничивает поиск релевантными объектами) - иначе локальный поиск был слишком медленным.

Локальный поиск: 
обмен клиентами между объектами, замена одного открытого объекта на закрытый поблизости, открытие нового объекта если экономия перекрывает стоимость открытия

### Мультистарт

Вместо одного жадного старта - много случайных стартов. Выделили 10% времени на построение начального решения, остальное - на локальный поиск.

### Возмущение

Если локальный поиск не улучшает решение, восстанавливаем лучшее найденное решение и закрываем несколько объектов с наименьшим числом клиентов.

Потом поменял жадник на первом старте на такую эвристику: на каждом шаге назначаем того клиента, у которого наибольший разрыв между стоимостью первого и второго самого подходящего объекта. Клиенты где мало хороших вариантов обрабатываются первыми и мы быстрее сходимся.
