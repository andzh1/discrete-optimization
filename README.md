## Описание алгоритма

- Жадник: на каждом шаге выбирается множество с лучшим соотношением цена / полезность покрытия. Полезность учитывает не только количество новых покрытых элементов, но и их редкость: элементы, которые встречаются в меньшем числе множеств, имеют больший приоритет.

- После построения начального покрытия из него удаляются множества, которые не нужны для сохранения полного покрытия.  
   Это простой и безопасный способ сразу уменьшить стоимость.

- local search: решение улучшается малыми перестройками: заменой дорогого множества на более дешевую альтернативу без потери покрытия, лии добавлением полезного множества с последующей очисткой от ставших лишними дорогих множеств. 

- Несколько запусков: начальное решение строится несколько раз с небольшой рандомизацией, после чего выбирается лучший результат и дооптимизируется локальным поиском.  
   Это делает качество стабильнее, чем один детерминированный запуск.

Плюс есть if на размер входа, который определяет колво запусков алгоритма в зависимости от мощности

Запускать: ./run.sh test_name
